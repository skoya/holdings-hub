# Holdings Hub Prototype — Meridian Bank (fictional)

> **Simulation only — no real transactions or investment advice.**

A fictional, GSIB-neutral static SPA demonstrating how a global systemically
important bank could act as the authoritative orchestration and trust hub
across traditional finance, regulated digital assets, tokenised finance and
(opt-in) DeFi. The institution shown — **Meridian Bank** — is fictional; no
affiliation with any real bank.

**Live:** https://skoya.github.io/holdings-hub/

The prototype never executes a real trade or transfer, never connects a
wallet, and never touches keys or production credentials. All identifiers are
deliberately fictional (`SIM-` prefixed LEIs, invalid-by-design IBANs).

## Stack

React 18 · Vite 7 · TypeScript (strict) · Tailwind 3 · Zustand · Zod ·
`pure-rand` (seeded deterministic engine) · localforage · Radix UI ·
`@xyflow/react` (process diagrams) · D3 (entity graph) · i18next · jszip ·
React Router (`HashRouter`) · Vitest 4 · Playwright — see `docs/adr/` for
decision records (the Vite 7 / Vitest 4 uplift from the plan's Vite 5 baseline
is logged in `docs/DECISIONS.md`).

## Develop

```bash
pnpm install
pnpm dev                 # local dev server
pnpm test:unit           # vitest — engine, schemas, persistence, import fuzz
pnpm test:integration    # vitest — wizard, holdings, transactions, round-trip
pnpm check:schemas       # config JSON validated against Zod
pnpm check:i18n          # translation key parity across locales
pnpm check:licenses      # production-dependency licence allowlist
pnpm lint && pnpm typecheck && pnpm format:check
pnpm build               # static build in dist/ (CSP meta injected)
pnpm e2e                 # Playwright (Chromium + WebKit); CI runs the full suite
```

Node ≥ 20, pnpm 9. E2E requires Playwright browser system libraries; in this
project the full E2E matrix runs in GitHub Actions.

## Security

- **Static site, no backend, no secrets.** Nothing to leak; no server trust
  boundary.
- **Content Security Policy** injected into the production build (`self` +
  the CoinGecko origin for the optional live-price overlay only).
- **Import is the only untrusted input** — strictly Zod-validated with a 5 MB
  size cap; hostile/malformed payloads and prototype-pollution attempts are
  rejected (see `tests/schemas/fuzz.test.ts`).
- **CI gates**: `audit-ci` (moderate+) on every push/PR; a weekly `security.yml`
  runs `pnpm audit`, the licence allowlist and the Google OSV scanner over
  `pnpm-lock.yaml`.
- **No `Math.random()`** anywhere in `src/` (lint-enforced); all randomness is
  seeded and reproducible.

## Determinism

All simulated randomness flows through a seeded engine
(`src/engine/prng.ts`); `Math.random()` is banned by lint rule. Same seed +
same actions ⇒ identical holdings, screening outcomes, ids and audit log.
Engine state serialises into the exported session JSON.
