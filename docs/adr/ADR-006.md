# ADR-006 — i18n: bundled `i18next` translations

Status: Accepted (2026-07-17)

Bundling de/fr/ja JSON at build time keeps the site fully static, offline from any translation API at runtime, and lets CI enforce key parity deterministically. Runtime translation services add a network dependency, a key/secret, and nondeterministic copy — all wrong for a static, deterministic prototype. Machine-generated quality is pre-accepted (PLAN Section 44); the build script makes regeneration one command.
