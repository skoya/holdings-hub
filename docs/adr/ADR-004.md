# ADR-004 — PRNG: `pure-rand` (over `seedrandom`)

Status: Accepted (2026-07-17)

The engine must save and restore generator state inside session JSON to guarantee replay determinism. `pure-rand` provides pure, immutable generators (xoroshiro128+) whose behaviour is deterministic and well-maintained (it underpins fast-check). `seedrandom` is stateful, encourages global patching, has no maintained releases in years, and makes state export awkward. This is the one substitution from the suggested stack.

Implementation note (2026-07-18): serialisation stores stream seeds + exact draw counts and replays on restore, rather than exporting raw generator state — version-proof against pure-rand internal changes. See docs/DECISIONS.md.
